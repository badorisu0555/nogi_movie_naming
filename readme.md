# Nogizaka46 Video Labeling API

## 📝 目的
乃木坂46の初見ユーザーが、SNS（Instagramリール、TikTok）上の動画を通じて「誰が誰か」を即座に判別できるようにする。
これにより初見ユーザーの認知負荷を下げ、関連コンテンツへの遷移率（ファン化率の先行指標）を現状の推定5%から15%以上へ向上させることを目的とする。

## 🎯 要求・ゴール (KPI設定)
* **API実装**: 動画ファイル（最大容量50MB / 最大長60秒）をリクエストとし、メンバー名と信頼スコア（0.00〜1.00）が描画された動画ファイルまたはURLを返す。
* **識別精度**: 主要メンバーの顔認識におけるF1スコア0.95以上を達成する。誤検知率（False Positive Rate）は5%未満とする。
* **処理レイテンシ**: 15秒の動画（約450フレーム）に対し、リクエストからレスポンスまでの処理時間を10秒以内に完了させる（必要スループット: 45fps以上）。
* **UI/UX向上**: 視認性を損なわないよう、バウンディングボックスの透過度を30%、フォントサイズを画面縦幅の5%に設定してオーバーレイ表示する。

## ⚙️ 要件定義

システムの処理を「動画解析（推論）」と「結果配信（APIレスポンス）」の2フェーズに分割します。

| 項目 | 内容 | 定量要件・制約 |
| :--- | :--- | :--- |
| **1. 動画解析フェーズ** | **推論処理 (非同期実行)** | |
| **入力データ** | 動画ファイル (mp4, mov) | 1ファイル最大50MB, 解像度1080p以下 |
| **解析パイプライン** | フレーム抽出 → 顔検知 → 特徴量抽出 → 照合 → 描画 | 全フレームではなく、3フレーム/秒 (3fps) で間引き処理を実行 |
| **メンバーDB** | 現役メンバーの学習済みモデル（Embeddingベクトル） | 1名あたり最低500枚の多様なアングル・照明の画像データを学習 |
| **出力作成** | 名前とスコアを重畳した動画のエンコード | 圧縮率(CRF)23でH.264エンコード |
| | | |
| **2. API配信フェーズ** | **APIリクエスト時 (REST API)** | |
| **スループット** | 同時リクエスト処理能力 | ピーク時100リクエスト/分を許容 |
| **可用性 (SLA)** | システム稼働率 | 99.9%以上 |
| **格納先** | S3 (一時ストレージ) | ストレージコスト削減のため、処理後24時間で自動削除 (TTL設定) |
| **出力形式** | JSONレスポンス | ペイロードサイズ 10KB以内 |

## 🏗️ システムアーキテクチャ

### 解析・配信フロー
```text
ユーザー (APIクライアント)
│
│ 1. リクエスト: 動画アップロード (Max 50MB)
▼
[API Gateway / FastAPI] (受付レイテンシ: < 100ms)
│ 2. S3へ動画保存
│ 3. 解析ジョブをキューに追加 (AWS SQS)
▼
[解析ワーカー (GPU Instance: NVIDIA T4 / A10G等)]
│ 4. 動画からフレーム抽出 (3fpsに間引き)
│ 5. 推論: 顔検知 & メンバー識別 (推論時間: < 50ms/frame)
│ 6. 描画: 名前 & スコアを描画 (OpenCV VideoWriter)
▼
[S3 Bucket] 🗄️ (結果動画保存, 24h TTL)
│
│ 7. 完了通知 / レスポンス返却 (Total Latency < 10s)
▼
ユーザー
```
